Консоль - единое место для ввода и вывода текстов в проекте Оператор.
Все взаимодействие с пользователем идет через эту консоль.

Классы, работающие с консолью, вынесены в пространство имен Lexicon.

Для сообщений консоли введена указанная ниже цветовая схема. 
Она должна соблюдаться и в коде самого Оператора, и в сборках процедур.
Для соблюдения этого правила весь вывод в статическую C# Console из кода запрещен.
Вместо этого должен использоваться объект Engine.OperatorConsole.
Его функции осуществляют соблюдение введенной цветовой схемы.

Процедуры должны использовать диалоги для ввода и вывода сложных данных.
Диалоги для алгоритма процедуры должны быть предварительно разработаны и написаны текстом, близким к естественному диалогу.
Диалоги состоят из Сообщений. Сообщения разделяются на классы и визуально различаются цветом текста. 
 Это упрощает восприятие содержимого консоли.

Классы сообщений:
- ввод Пользователя
- Предупреждение
- Вопрос
- Подтверждение
- Сообщение

Цветовая схема сообщений:
- цвета текстов сообщений: енум DialogConsoleColors
  - ВводПользователя = ConsoleColor.White, Цвет текстов, введенных пользователем. 
    - Визуально отделяет тексты пользователя от всех остальных, облегчает восприятие смысла текста в консоли.
  - Предупреждение = ConsoleColor.DarkMagenta, Цвет выводимых в консоли предупреждений.
    - Этим цветом должны обозначаться все предупреждения, сообщения об исключениях, нефатальные ошибки процессов Оператора.
  - Вопрос = ConsoleColor.DarkYellow, Цвет выводимых в консоли вопросов пользователю.
    - Этим цветом должны обозначаться все запросы/вопросы к пользователю.
  - Успех = ConsoleColor.DarkGreen, Цвет выводимых в консоли подтверждений об успехе.
    - Этим цветом должны обозначаться все сообщения об успехах.
  - Сообщение = ConsoleColor.Gray, Цвет для всех прочих текстов, выводимых пользователю.
    - Этим цветом выводятся все тексты, не относящиеся к вышеуказанным категориям. 


Поясняющие описания-подсказки в диалогах должны выводиться перед запросом ввода пользователя, в виде пунктов: 
" - Текст пункта, однострочный." и 
"   Продолжение текста пункта." и 
"   Пример: Текст пункта."
Такие подсказки предоставляют пользователю несложную информацию в процессе диалога.
Для сложной и объемной информации придется явно отсылать пользователя к руководству,
 традиционным старым книжным способом, поскольку веб-ссылок в речевом интерфейсе не может быть.
